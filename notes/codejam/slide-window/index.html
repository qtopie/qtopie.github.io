滑动窗口双指针算法详解

October 6, 2024

什么是滑动窗口？

滑动窗口算法是一种常用的算法技巧，尤其适用于数组或字符串上的子序列问题。它通过两个指针（通常称为 left 和 right）来定义一个窗口，窗口的大小可以动态调整。这两个指针在数组或字符串上滑动，从而枚举出所有的可能子序列。

核心思想：

  • 窗口的定义: 两个指针 left 和 right 定义了一个窗口。
  • 窗口的滑动: right 指针向右移动，窗口扩大；当窗口不满足条件时，left 指针向右移动，窗口缩小。
  • 问题的解决: 在窗口滑动的过程中，不断判断窗口是否满足题目要求，并更新答案。

时空置换, 一般会存储窗口内的结果,来预测窗口变换的结果. 固定left, 移动right尝试所有可能的解, 如果不行,则移动right到下一个. 相对于暴力算法,对一同一个left的结果集,可以避免很多无效的枚举

滑动窗口算法的适用场景

  • 寻找最长/最短的符合条件的子数组/子字符串: 例如，寻找最长连续子数组的和不超过某个值。
  • 判断一个字符串是否包含另一个字符串的所有字符: 例如，判断一个字符串是否包含另一个字符串的所有字符，且每个字符出现的次数都不少于目标字符串中出现的次数, 输出其最短字串的长度。
  • 求数组/字符串中所有满足条件的子数组/子字符串的个数: 例如，求数组中所有和为 k 的子数组的个数。

滑动窗口算法的模板

def sliding_window(nums, k):
    left = 0
    right = 0
    # 初始化窗口内的元素的计算
    window_sum = 0
    # 在这里写你想要跟踪的变量，比如最大值，最小值等
    max_sum = float('-inf')

    while right < len(nums):
        # 将新元素加入窗口
        window_sum += nums[right]
        right += 1

        # 判断窗口是否需要缩小
        while left < right and window_sum > k:
            # 从窗口中移除元素
            window_sum -= nums[left]
            left += 1

        # 在这里更新答案
        max_sum = max(max_sum, window_sum)

    return max_sum

滑动窗口算法的优化

  • 数据结构的选择: 对于需要快速查询元素是否存在的场景，可以使用哈希表来优化。
  • 窗口大小的限制: 如果题目对窗口大小有明确限制，可以提前进行判断，避免不必要的计算。
  • 单调队列: 在某些情况下，可以使用单调队列来优化滑动窗口，例如求滑动窗口内的最大值或最小值。

总结

滑动窗口算法是一种非常灵活的算法，可以解决很多类型的数组和字符串问题。它的核心思想是通过两个指针来维护一个动态的窗口，并不断地更新窗口的状态。在使用滑动窗口算法时，关键是要明确窗口的含义、滑动条件以及需要维护的变量。

注意: 滑动窗口算法的具体实现方式会根据不同的问题而有所不同，但其基本思想是相同的。
